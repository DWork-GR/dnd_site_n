# Как самостоятельно менять сайт

## Где что находится

- `index.html` — разметка мастерской части сайта и её диалоговые окна.
- `player.html` — разметка личного кабинета игрока.
- `assets/css/base.css` — общие цвета, кнопки, навигация и базовые стили мастера.
- `assets/css/pages.css` — карточки и основные страницы мастера.
- `assets/css/character-sheet.css` — листы персонажей и досье NPC.
- `assets/css/player.css` — оформление личного кабинета игрока.
- `assets/js/core.js` — общие данные, нормализация старых сохранений и вспомогательные функции.
- `assets/js/views.js` — отрисовка страниц и карточек мастера.
- `assets/js/sheets.js` — открытие и заполнение редакторов.
- `assets/js/events.js` — обработчики кнопок, кликов и сохранения форм.
- `assets/js/main.js` — запуск мастерской части и синхронизация с сервером.
- `assets/js/player.js` — вся логика личного кабинета игрока.
- `server/index.js` — API, авторизация, база данных и фильтрация приватной информации.

## Обычный порядок изменения функции

Например, чтобы добавить новое поле NPC:

1. Добавьте поле формы в `index.html`.
2. Заполните поле при открытии редактора в `openNpcDossier()` из `assets/js/sheets.js`.
3. Сохраните поле в обработчике `#npc-form` из `assets/js/events.js`.
4. Покажите поле в карточке или досье через `assets/js/views.js` или `assets/js/sheets.js`.
5. Если поле должны видеть игроки, явно добавьте безопасную версию в `publicNpc()` из `server/index.js`.

Последний пункт важен: не отправляйте игрокам мастерские заметки, секреты и скрытые характеристики.

## Команды после изменений

Привести HTML, CSS и JavaScript к единому читаемому виду:

```powershell
npm.cmd run format
```

Проверить форматирование без изменения файлов:

```powershell
npm.cmd run format:check
```

Проверить синтаксис JavaScript:

```powershell
npm.cmd run check
```

Запустить сайт:

```powershell
npm.cmd start
```

## Советы

- Для цвета интерфейса сначала ищите CSS-переменные в `assets/css/base.css`.
- Для кнопки или карточки ищите её `id`, класс или `data-*` атрибут через поиск по всему проекту.
- Не редактируйте `package-lock.json`, `node_modules/` и файлы в `uploads/` вручную.
- После изменения серверного кода перезапускайте сайт.
- Перед коммитом запускайте `npm.cmd run format` и `npm.cmd run check`.
